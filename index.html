Benchmark Games for Gerbil, Racket, Go, and C (GCC)

This is based on the now defunct Programming Language Shootout, aka The Benchmark Games.

I run the tests with Gerbil, and compared with Racket, Go, and C. The Gerbil programs are new, for the others I used the existing (expertly written and heavily optimized) programs from the Language Shootout Archive.

Versions used: Gerbil v0.18, Racket v8.2, Go 1.21.2, and GCC 11.4

The source code for the benchmark game is here.

The benchmarks were run on a Dell XPS 13-9320 laptop.

Each benchmark was run 3 times, and I kept the best value.

Note C programs were compiled with gcc -O2.

Note Some benchmarks have a fixed small input, and take 0 time to complete (less than the 10ms resolution of /usr/bin/time); so they are not included here, as they are pointless.

Note With the exception of chameneosredux and threadring, the other benchmark all utilize a single core. I want to compare apples to apples, not how well a given language can multicore.

Note In the chameneosredux, Go and GCC produce non-deterministic output.

NoteThis is not a d*ck swinging contest. I compiled these results to see how far I can push Gerbil's performance envelope and evaluate progress as the macrology/compiler improves. Take them with a grain of salt.
